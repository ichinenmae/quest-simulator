# Codex向け開発依頼

## 依頼概要

GitHub Pagesで動作する、フードデリバリー追加報酬比較・週間稼働計画シミュレーターを実装してください。

対象環境：

- Windows Chrome
- Android Chrome
- GitHub Pages 無料プラン

制約：

- サーバーなし
- 外部DBなし
- ユーザー登録なし
- OCRなし
- AI API連携なし
- LocalStorage保存
- JSONインポート/エクスポート対応

## 実装方針

最初は静的Webアプリとして実装してください。  
React等のビルド環境を使う場合は、GitHub Pagesにデプロイしやすい構成にしてください。  
単一HTML/CSS/JS構成でも構いません。

## 必須機能

### 1. サービス設定

各サービスについて以下を設定できる。

- サービス名
- 平均基本報酬/件
- 平均件数/h
- 有効/無効

初期値例：

- Uber
- 出前館
- Wolt
- menu

### 2. クエスト入力

以下に対応。

- AI抽出テキスト貼り付け
- 手入力
- 固定マイルストーン
- 継続ボーナス
- 期間クエスト
- 時間帯クエスト
- 荒天クエスト
- 終日追加報酬
- その他

### 3. AI抽出テキスト解析

`AI_IMPORT.md` の形式に対応。

要件：

- 貼り付け欄
- 解析
- バリデーション
- 登録前プレビュー
- 手動修正
- 登録確定

### 4. 週間計画

曜日ごとに稼働予定を設定できる。

- 稼働ON/OFF
- 複数時間帯
- 件数自動計算
- 件数手動上書き
- 先週コピー
- テンプレート保存・適用

### 5. 期間クエスト選択支援

候補ごとに以下を表示。

- 目標件数
- 報酬
- 必要稼働時間
- 予想件数
- 達成見込み
- 達成時売上
- 達成時時給
- 追加必要件数
- 追加必要時間

推奨目標を表示。

### 6. サービス比較

指定時間帯について、各サービスの実質単価・実質時給を比較する。

さらに、Uber期間クエストへの影響を表示する。

### 7. 実績・進捗

- 現在件数入力
- 現在売上入力
- 残り必要件数
- 残り必要時間
- 達成見込み
- 進捗バー

### 8. 推奨休業日

曜日ごとに「休んだ場合」をシミュレーションし、休業候補をランキング表示する。

### 9. データ保存

- LocalStorage保存
- JSONエクスポート
- JSONインポート
- データ初期化

### 10. スクリーンショット共有向け表示

LINEに貼りやすい縦長カード形式の結果表示を用意する。

対象：

- 週間予測
- 推奨期間クエスト
- サービス比較
- 推奨休業日
