# 計算ルール

## 1. 基本

### 1.1 予想件数

```text
予想件数 = 稼働時間 × 件数/h
```

手動上書きがある場合：

```text
予想件数 = 手動上書き件数
```

### 1.2 基本報酬

```text
基本報酬 = 予想件数 × 平均基本報酬/件
```

### 1.3 クエスト込み売上

```text
総売上 = 基本報酬 + 達成見込みクエスト報酬
```

概要画面では、選択済みまたは推奨中の期間クエスト報酬に加え、時間帯・荒天・終日・その他クエストの達成見込み報酬をすべて含める。

月〜木と金〜日の期間クエストは、それぞれの対象期間内の予想件数で個別に達成判定し、達成見込み報酬を合算する。時間帯などのその他クエストは対象曜日・日付・時間帯と稼働枠が重なる件数だけで判定し、週売上には1クエストにつき1回だけ加算する。

### 1.4 予想時給

```text
予想時給 = 総売上 ÷ 稼働時間
```

---

## 2. マイルストーン報酬

対象件数以下で達成可能な最大マイルストーンを採用する。

例：

| マイルストーン | 報酬 |
|---:|---:|
| 3 | 300 |
| 6 | 600 |
| 12 | 1500 |

予想件数 7件の場合：

```text
報酬 = 600
```

---

## 3. 継続ボーナス展開

固定マイルストーン：

| count | reward |
|---:|---:|
| 1 | 200 |
| 2 | 300 |
| 3 | 600 |

継続：

- 4件目から20件目
- 1件ごと +200円

展開：

| count | reward |
|---:|---:|
| 4 | 800 |
| 5 | 1000 |
| 6 | 1200 |

計算式：

```text
reward(count) = baseReward + (count - startCount + 1) × bonusPerDelivery
```

baseReward は startCount 未満の最大マイルストーン報酬。

---

## 4. ボーナス重複

同じ配達が複数クエストの対象になる場合、件数は共有されるが、報酬は加算する。

例：

- 期間クエスト対象件数：80件
- 時間帯クエスト対象件数：12件

この12件は80件の内数。  
総件数を 92件にはしない。

---

## 5. 期間クエスト選択

各候補について以下を計算する。

### 5.1 必要稼働時間

```text
必要稼働時間 = 目標件数 ÷ 件数/h
```

### 5.2 達成時基本報酬

```text
達成時基本報酬 = 目標件数 × 平均基本報酬/件
```

### 5.3 達成時売上

```text
達成時売上 = 達成時基本報酬 + 期間クエスト報酬
```

### 5.4 達成時時給

```text
達成時時給 = 達成時売上 ÷ 必要稼働時間
```

目標選択画面では期間クエスト候補同士の比較を主目的とするため、時間帯・荒天・終日・その他クエストは主売上・主時給に含めず、名称付きの補助情報として別表示する。

### 5.5 追加必要件数

```text
追加必要件数 = max(0, 目標件数 - 予想件数)
```

### 5.6 追加必要時間

```text
追加必要時間 = 追加必要件数 ÷ 件数/h
```

---

## 6. 達成見込み表示

目安：

| 表示 | 条件 |
|---|---|
| ◎ | 予想件数 >= 目標件数 + 余裕件数 |
| ○ | 予想件数 >= 目標件数 |
| △ | 追加必要時間が許容範囲内 |
| × | 達成困難 |

余裕件数や許容追加時間は設定で調整できるとよい。

---

## 7. サービス比較

指定時間帯に対して、各サービスごとに以下を計算する。

### 7.1 時間帯予想件数

```text
時間帯予想件数 = 時間帯の稼働時間 × サービス別件数/h
```

手動上書き可能。

### 7.2 実質単価

```text
実質単価 = (基本報酬 + ボーナス報酬) ÷ 予想件数
```

### 7.3 実質時給

```text
実質時給 = 実質単価 × 件数/h
```

または

```text
実質時給 = (基本報酬 + ボーナス報酬) ÷ 稼働時間
```

### 7.4 期間クエスト影響

他社へ切り替えた場合、Uberの期間クエスト予想件数から、その時間帯のUber予定件数を減算する。

減算後に選択中の期間クエストが未達になる場合、損失見込みとして期間クエスト報酬を表示する。

---

## 8. 推奨休業日

各曜日について、その日を休みにした場合の再計算を行う。

### 8.1 評価項目

- 売上減少額
- 時給低下
- 期間クエスト達成可否
- 時間帯クエスト喪失
- 残り日で挽回可能か

### 8.2 推奨順位

優先条件：

1. 期間クエスト未達にならない
2. 売上減少が小さい
3. 予想時給が低い
4. 時間帯クエストが少ない

---

## 9. 実績入力後の再計算

### 9.1 残り必要件数

```text
残り必要件数 = max(0, 目標件数 - 実績件数)
```

### 9.2 残り必要時間

```text
残り必要時間 = 残り必要件数 ÷ 件数/h
```

### 9.3 最終予想件数

```text
最終予想件数 = 実績件数 + 残り予定件数
```

### 9.4 達成予測

```text
最終予想件数 >= 目標件数
```

なら達成見込みあり。

---

## 10. 表示上の丸め

- 金額：整数円、カンマ区切り
- 時給：整数円/h
- 件数：小数1桁まで許容
- 時間：小数1桁まで、または「2時間30分」形式
