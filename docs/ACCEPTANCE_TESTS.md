# 受け入れテスト

## 1. AI抽出テキスト入力

### ケース1：期間クエスト

入力：

```text
QUEST_TYPE: PERIOD
SERVICE: Uber
TITLE: 月〜木 期間クエスト
PERIOD: MON_THU

60=3160
80=4340
100=5680
```

期待：

- 期間クエストとして認識される
- 候補3件がプレビューされる
- 登録後、期間クエスト選択画面に表示される

### ケース2：荒天クエスト

入力：

```text
QUEST_TYPE: WEATHER
SERVICE: Uber
TITLE: 雨クエスト
DAY: MON
START: 17:00
END: 21:30

1=200
2=300
3=600

REPEAT_START=4
REPEAT_END=20
REPEAT_BONUS=200
```

期待：

- 1〜20件までの累計報酬を計算できる
- 20件時点で4000円になる

## 2. 週間計画

条件：

- Uber 件数/h = 3.5
- 月〜木で合計20時間稼働

期待：

```text
予想件数 = 70件
```

手動上書きで80件を入力した場合：

```text
予想件数 = 80件
```

---

## 3. 期間クエスト選択

条件：

- Uber 基本報酬 = 550円/件
- 件数/h = 3.5
- 予想件数 = 80件
- クエスト候補：
  - 60件 3160円
  - 80件 4340円
  - 100件 5680円

期待：

- 60件と80件は達成見込み
- 100件は追加必要件数20件
- 100件の追加必要時間は約5.7時間
- 推奨目標が表示される

---

## 4. ボーナス重複

条件：

- 月〜木期間クエスト 80件 4340円
- 月曜昼 時間帯クエスト 12件 1500円
- 月曜昼に12件配達
- 月〜木合計80件

期待：

- 総件数は80件
- クエスト報酬は4340 + 1500 = 5840円
- 12件を期間件数と時間帯件数に二重計上しない

---

## 5. サービス比較

条件：

- Uber 実質時給 2400円
- 出前館 実質時給 2600円
- ただし出前館に切り替えるとUber期間クエストが未達になる

期待：

- 出前館の方が時給上位として表示される
- ただしUber期間クエスト未達リスク警告が出る
- 損失見込みが表示される
