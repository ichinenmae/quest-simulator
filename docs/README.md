# フードデリバリー追加報酬比較・週間稼働計画シミュレーター 開発資料

## 目的

Uber Eats を中心としたフードデリバリーの追加報酬（クエスト・ボーナス）を入力し、以下を判断できる Web アプリを作る。

1. 来週どのくらい稼働するべきか
2. Uber の期間クエストでどの目標件数を選ぶべきか
3. 同じ時間帯に他社ボーナスがある場合、どのサービスで配達するべきか
4. 休みにするならどの日がよいか
5. 週中・当日に、実績を入力して残り必要件数や達成見込みを確認する

## 利用環境

- Windows Chrome
- Android Chrome
- GitHub Pages 無料プラン
- サーバーなし
- ユーザー登録なし
- クラウド同期なし

## 想定ユーザー

- フードデリバリー配達員
- Uber Eats、出前館、Wolt、menu などを併用する人
- 友人知人にも URL を共有して使ってもらう想定

## 入力方式

1. AI抽出テキスト貼り付け
   - Gemini / ChatGPT に Uber アプリのスクリーンショットを読ませる
   - 指定フォーマットで出力
   - Web アプリに貼り付けて登録
2. 手入力
   - AI読み取り結果の修正にも使う
3. JSONインポート
   - 本人用バックアップ・復元用

## 共有方式

- JSON共有は主目的ではない
- LINEなどに貼れるスクリーンショット共有を重視
- 比較結果・週間予測・推奨休業日などを画像化できるとよい

## 除外する機能

初期版では以下を実装しない。

- OCR機能
- 画像認識機能
- チップ計算
- エリア別設定
- GPS
- アカウント機能
- クラウド同期
- 実績自動取得

## Phase 1 実装の起動方法

ビルドは不要。GitHub Pagesではリポジトリ直下をそのまま公開する。

ローカル確認：

```powershell
node scripts/serve.mjs
```

ブラウザで `http://localhost:4173` を開く。

自動テスト：

```powershell
node --test
```
