# スクリーンショットOCR用プロンプト

Uberなどのクエスト画面をChatGPTへ添付し、SumuQuestの「AIテキスト」へ貼り付ける形式に変換するためのプロンプト。

## 使用方法

1. 同じ週のクエスト画面を、上から下まで重複を含めてスクリーンショットする。
2. ChatGPTへ画像と下記プロンプトを送る。
3. 出力結果だけをSumuQuestの「AIテキスト」へ貼り付ける。
4. 登録前プレビューで日付、時間、件数、累計報酬を確認する。

画像に年が表示されない場合は、プロンプト冒頭の対象年を入力する。年を確認できない場合は空欄のままにし、AIに推測させない。

## ChatGPTへ渡すプロンプト

```text
添付画像はフードデリバリーサービスのクエスト画面です。
画像をOCRし、SumuQuestへ貼り付けられるAIテキストへ変換してください。

対象サービス: Uber
対象年: [例: 2026。確認できない場合は空欄]

次のルールを厳守してください。

【最重要】
- 出力は指定フォーマットだけにする。説明、前置き、Markdownのコードフェンス、箇条書きの解説は出力しない。
- 画像で確認できる情報だけを使い、数字・曜日・年・時刻を推測しない。
- 読めない必須項目は UNKNOWN とする。
- 同じ画面を重複して撮影した画像があっても、同一クエストを重複出力しない。
- 金額から円記号、プラス記号、カンマを除去する。
- 時刻は24時間表記の HH:MM にする。午前10時30分は10:30、午後5時は17:00とする。
- 日付はYYYY-MM-DDにする。画像に年がなく、対象年も未指定ならUNKNOWNとする。

【報酬の変換】
- SumuQuestの「件数=報酬」は、必ずその件数まで達成したときの累計報酬にする。
- 画面の1段目が「3回 300円」、2段目が「+3回 +400円」、3段目が「+3回 +500円」なら、次のように変換する。
  3=300
  6=700
  9=1200
- 「+N回」は直前までの累計件数へNを加える。
- 「+金額」は直前までの累計報酬へ加える。追加額をそのまま累計報酬として出力しない。
- 各行が最初から累計件数・累計報酬で表示されている場合は、その数値をそのまま使う。
- 「拒否回数が○回未満」などは達成件数や報酬ではない。件数=報酬の行やREPEAT_*へ変換しない。
- エリア、拒否回数、完了した配達のみ対象など、現在の出力形式に項目がない条件は出力しない。

【期間クエスト】
- 月曜日04:00から金曜日04:00はPERIOD: MON_THUとする。
- 金曜日04:00から月曜日04:00はPERIOD: FRI_SUNとする。
- 月曜日04:00から金曜日04:00と表示されている場合、START_DATEは月曜日、END_DATEは木曜日の日付にする。
- 金曜日04:00から月曜日04:00と表示されている場合、START_DATEは金曜日、END_DATEは日曜日の日付にする。
- 選択候補として「クエスト1」「クエスト2」などが並ぶ場合、それぞれを別クエストにせず、1つのPERIODブロック内の候補行としてまとめる。
- 各「クエストN」カードの最初の黒字の件数・緑字の報酬だけを、選択候補として出力する。
- 鍵アイコンが付いた灰色の「+10回の乗車」「+金額」などの追加行は、現在のSumuQuestの期間目標候補には出力しない。
- 例えばクエスト1が「120回 19,840円」、クエスト8が「50回 4,150円」なら、同じPERIODブロックへ次のように出力する。
  50=4150
  120=19840
- 別の選択候補も同じPERIODブロックへ件数順に追加する。
- 同じ件数の候補が複数ある、または件数と報酬の組み合わせを確実に読めない候補は出力しない。推測で補完しない。

【時間帯・荒天クエスト】
- 1つの日付・時間帯ごとに1ブロック作る。
- 同じ曜日でも昼枠と夜枠は別ブロックにする。
- 通常の時間指定プロモーションはQUEST_TYPE: TIMEとする。
- 画像や説明に雨、雪、悪天候、荒天などの条件が明記されている場合だけQUEST_TYPE: WEATHERとする。
- 時間帯が04:00から翌03:59までの終日条件ならQUEST_TYPE: DAILYとする。
- それ以外で分類不能の場合だけQUEST_TYPE: OTHERとする。

【継続ボーナス】
- 一定件数以降、1件ごとに同じ金額が加算されると明記されている場合だけREPEAT_*を使う。
- 上限がないと明記されている場合はREPEAT_END=UNLIMITEDとする。
- 段階ごとに追加額が異なる通常クエストをREPEAT_*へ変換しない。

【出力形式】

期間クエスト:
QUEST_TYPE: PERIOD
SERVICE: Uber
TITLE: 月～木期間クエスト または 金～日期間クエスト
PERIOD: MON_THU または FRI_SUN または UNKNOWN
START_DATE: YYYY-MM-DD または UNKNOWN
END_DATE: YYYY-MM-DD または UNKNOWN

件数=累計報酬
件数=累計報酬

時間帯クエスト:
QUEST_TYPE: TIME
SERVICE: Uber
TITLE: 月昼時間帯クエスト など
DAY: MON/TUE/WED/THU/FRI/SAT/SUN または UNKNOWN
START_DATE: YYYY-MM-DD または UNKNOWN
END_DATE: YYYY-MM-DD または UNKNOWN
START: HH:MM または UNKNOWN
END: HH:MM または UNKNOWN

件数=累計報酬
件数=累計報酬

荒天クエストはQUEST_TYPE: WEATHER、終日はDAILY、分類不能はOTHERを使用する。

継続ボーナスがある場合のみ末尾へ追加する:
REPEAT_START=開始件数
REPEAT_END=終了件数 または UNLIMITED
REPEAT_BONUS=1件ごとの追加額

複数クエストは、ブロック間を次の1行で区切る:
---

可能であれば必ず --- を入れる。
ただし、--- を出力できない場合でも、各クエストは必ず QUEST_TYPE: から開始する。
SumuQuestは QUEST_TYPE: の出現も新しいクエストブロックとして扱う。

最後に自分の出力を内部確認し、次を満たしてから回答する。
- 件数が昇順である
- 累計報酬が減少していない
- +N回と+金額を累計へ変換した
- 同じクエストを重複出力していない
- 拒否回数を配達件数へ混ぜていない
- 読めない値を推測していない
```

## 動画で確認した変換例

画面表示：

```text
3回の乗車 350円
+3回の乗車 +450円
+3回の乗車 +600円
```

出力：

```text
3=350
6=800
9=1400
```

画面表示：

```text
6回の乗車 700円
+3回の乗車 +500円
```

出力：

```text
6=700
9=1200
```
